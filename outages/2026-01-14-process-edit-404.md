# Process edit route 404

## Summary
- `/processes/[id]/edit` returned a 404 because the edit route was not implemented in the Astro
  routes.

## Impact
- Users could not edit custom processes.
- Custom process editing fell out of parity with the inventory and quest editors, blocking QA
  workflows that rely on updates.

## Fix
- Added the edit route and shared process form wiring to load existing custom processes and
  persist updates to the same ID.
- Added an end-to-end regression test that validates the edit page renders, prefills data, and
  persists changes.

## Lessons
- Maintain route parity checks across custom content types (items, quests, processes).
- Add automated tests for edit links and edit-route rendering to catch missing routes.
- Keep create/edit flows on the same form component to prevent drift.
