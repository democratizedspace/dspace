# quest-form-finish-placeholder

- **Date:** 2025-10-30
- **Component:** frontend:quest-form
- **Root cause:** QuestForm appended new finish options even when the starter node still contained the autogenerated finish placeholder, so quests with a custom finish choice rendered two finish buttons and the quests Playwright run failed its single-option assertion.
- **Resolution:** When a user adds a finish option, QuestForm now replaces the autogenerated placeholder instead of appending. Added a regression unit test that verifies only the custom finish option remains and reran the quests end-to-end suite to confirm the behaviour.
- **References:**
  - frontend/src/components/svelte/QuestForm.svelte
  - frontend/src/components/__tests__/QuestForm.spec.ts
  - frontend/e2e/quests.spec.ts
