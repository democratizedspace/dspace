# DSPACE QA checklists

Use these checklists to keep releases stable and predictable. Each release gets
its own copy so we can track what changed without losing historical context.

- [v3 QA checklist](./v3)
- [v3.1 QA checklist (placeholder)](./v3.1.md) — token.place integration + rerun
  the v3 suite
- [Reusable template](./template.md)

## How to use this folder

- Copy the latest checklist for each new release (`v3.0.1`, `v3.1`, etc.).
- Keep a “Quick command index” block near the top of each checklist so teams can
  find the canonical automation fast.
- Keep release-specific notes and links inside the checklist you copy so
  historical expectations stay readable.
