## 2.1) `/quests` performance launch gate (required for TTI optimization work)

Primary validation stack (required): **DevTools trace review + User Timing milestones + Playwright harness**.
Lighthouse is supplementary and must not be used as the only sign-off artifact.

- [ ] Run the `/quests` Playwright harness on the staging candidate (or RC-equivalent env)
- [ ] Record timing output for these User Timing measures:
  - `quests:time-to-builtin-visible`
  - `quests:time-to-snapshot-ready`
  - `quests:time-to-full-reconciliation`
  - `quests:time-to-custom-merge`
- [ ] Confirm all `/quests` User Timing marks/measures exist and durations are sensible (no missing milestones)
- [ ] Capture and review at least one Chromium DevTools Performance trace for `/quests`
- [ ] Confirm the trace critical path shows meaningful built-in list visibility before full-state reconciliation
- [ ] Optional supporting audit: run Lighthouse locally and attach results (supplementary only)

Suggested commands:

```bash
cd frontend
QUESTS_PERF_CPU_THROTTLE=4 npm run perf:quests
```

```bash
# Optional supplementary audit (not sufficient by itself)
npx lighthouse http://localhost:3000/quests --only-categories=performance
```

Recordkeeping template:

- Candidate tag/SHA: `________________`

- Trace artifact link/path: `________________`

## 3) Staging deploy + validation

Deploy approved immutable candidate to staging (`staging.democratized.space`) and validate:

- [ ] `/config.json` returns expected runtime config
- [ ] `/healthz` and `/livez` return success
- [ ] Patch issue reproduction is fixed in staging
- [ ] No regression in core nav/routes relevant to touched areas
- [ ] QA Cheats remain enabled in staging (`DSPACE_ENV=staging`)

Suggested smoke commands:

```bash
curl -fsS https://staging.democratized.space/config.json
```

```bash
curl -fsS https://staging.democratized.space/healthz
```

```bash
curl -fsS https://staging.democratized.space/livez
```

---

## 4) Production promotion gate

- [ ] Promote only the exact immutable tag validated in staging
- [ ] Confirm rollback tag and command are prepared before prod deploy
- [ ] Confirm prod values file and host target are correct (`democratized.space`)
- [ ] Confirm QA Cheats remain OFF in prod (`DSPACE_ENV=prod`)

Suggested prod smoke commands:

```bash
curl -fsS https://democratized.space/config.json
```

```bash
curl -fsS https://democratized.space/healthz
```

```bash
curl -fsS https://democratized.space/livez
```

---

## 5) Rollback notes

If production verification fails:

- [ ] Roll back immediately to previous known-good immutable tag
- [ ] Re-run prod health checks
- [ ] Log incident notes and follow-up fix scope
- [ ] Keep `v3.0.0.1` open until replacement candidate is validated

Rollback command template:

```bash
cd ~/sugarkube
```

```bash
just helm-oci-upgrade release=dspace namespace=dspace chart=oci://ghcr.io/democratizedspace/charts/dspace values=docs/examples/dspace.values.prod.yaml version_file=docs/apps/dspace.version tag=<previous-immutable-tag>
```
