# DSPACE v3.0.0.1 QA Checklist (urgent patch line)

> Scope: urgent bugfixes and small improvements only.
>
> This checklist is intentionally slimmer than `docs/qa/v3.md` and is **not** for feature-track work.
> Keep feature-track planning in `docs/qa/v3.1.md`.

---

## 0) Release metadata

- [ ] Release type confirmed: urgent patch (no feature scope)
- [ ] Candidate branch: `<main or release/v3.0.0.1>`
- [ ] Candidate commit SHA: `<fill>`
- [ ] Candidate immutable image tag: `<fill>`
- [ ] Planned semver tag: `v3.0.0.1`
- [ ] Staging URL: `https://staging.democratized.space`
- [ ] Prod URL: `https://democratized.space`
- [ ] Rollback immutable tag recorded: `<fill>`

---

## 1) Patch-scope gate

- [ ] Every included PR is bugfix/small-improvement only.
- [ ] Deferred feature work moved to the `v3.1.x` track.
- [ ] Changelog/release notes describe user-visible fixes clearly.

---

## 2) Automated checks (required)

- [ ] `npm run lint`
- [ ] `npm run type-check`
- [ ] `npm run test:ci`
- [ ] `npm run link-check`
- [ ] `node run-tests.js` (or documented reason if intentionally skipped)

---

## 3) Staging validation (must pass before prod)

Deploy the chosen immutable candidate tag to staging using
[../k3s-sugarkube-staging.md](../k3s-sugarkube-staging.md).

- [ ] `GET /config.json` returns expected release metadata
- [ ] `GET /healthz` is healthy
- [ ] `GET /livez` is healthy
- [ ] Reproduced bugfix scenarios are verified
- [ ] No regressions in touched areas
- [ ] QA Cheats behavior in staging remains correct (**ON**)

Recommended targeted manual smoke:

- [ ] Home, Quests, Inventory, Docs, Chat routes load
- [ ] Any touched flows in this patch line are tested end-to-end
- [ ] Browser console has no new critical errors on touched pages

---

## 4) Production promotion gate

- [ ] Staging sign-off recorded with approver + date
- [ ] Same immutable tag selected for prod deployment
- [ ] Deployment command and tag captured in release notes
- [ ] Prod deploy executed via [../k3s-sugarkube-prod.md](../k3s-sugarkube-prod.md)
- [ ] Prod checks pass: `/config.json`, `/healthz`, `/livez`
- [ ] QA Cheats behavior in prod remains correct (**OFF**)

---

## 5) Rollback notes

If production checks fail, redeploy the pre-recorded rollback immutable tag immediately.

- [ ] Rollback command prepared before prod deploy
- [ ] Rollback tag previously verified in staging or prior prod
- [ ] Post-rollback health checks pass
- [ ] Incident/release notes updated with what failed and follow-up actions

---

## 6) Finalize release

- [ ] Create/push semver tag `v3.0.0.1` on approved commit
- [ ] Publish release notes with fixed issues and rollback reference
- [ ] Confirm follow-up items moved to `v3.1.x` if feature-sized
