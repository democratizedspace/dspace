# DSPACE v3.0.0.1 QA / Release Checklist (Patch Lane)

> Scope: urgent bugfixes and small improvements only.
>
> This checklist is intentionally slimmer than the v3.0.0 launch checklist. Use it for patch-level
> releases where release risk is managed by targeted validation plus core smoke checks.

## 1) Release metadata

- [ ] Release version: `v3.0.0.1`
- [ ] Release owner:
- [ ] QA owner:
- [ ] Target date:
- [ ] Candidate immutable tag (example `main-abc1234`):
- [ ] Previous known-good prod tag:
- [ ] Linked PR(s):
- [ ] Linked issue(s):
- [ ] Patch scope summary (bugs fixed / small improvements):

## 2) Scope guard (patch discipline)

- [ ] Confirms **no feature-track scope** (new feature work belongs in `v3.1.x` track)
- [ ] Confirms changes are bugfixes/small improvements only
- [ ] Confirms changelog entry is accurate and minimal
- [ ] Confirms migration risk is none or explicitly documented

## 3) Candidate preparation

- [ ] Candidate tag selected as immutable (`main-<shortsha>` or `v3.0.0.1-rc.N`)
- [ ] Candidate deployed to staging (`staging.democratized.space`)
- [ ] Staging environment kept as `DSPACE_ENV=staging` (QA Cheats ON)

Reference runbooks:

- [Staging runbook](../k3s-sugarkube-staging.md)
- [Prod runbook](../k3s-sugarkube-prod.md)
- [Release management policy](../merge-plan.md)

## 4) Staging verification gates

### 4.1 Health + availability

- [ ] `/config.json` returns 200 on staging
- [ ] `/healthz` returns 200 on staging
- [ ] `/livez` returns 200 on staging

### 4.2 Patch-targeted verification

- [ ] Repro steps for each fixed bug pass in staging
- [ ] No regression in directly adjacent workflows
- [ ] Any updated docs match actual behavior

### 4.3 Core smoke (minimum)

- [ ] Home page loads
- [ ] Primary navigation works
- [ ] Quests, Inventory, Docs, Chat pages load
- [ ] Console/network check shows no release-blocking errors

## 5) Production promotion gate

- [ ] Promotion approved from staging results
- [ ] Exact same immutable tag scheduled for prod
- [ ] Prod environment remains `DSPACE_ENV=prod` (QA Cheats OFF)

## 6) Production verification

- [ ] `/config.json` returns 200 on `https://democratized.space`
- [ ] `/healthz` returns 200 on `https://democratized.space`
- [ ] `/livez` returns 200 on `https://democratized.space`
- [ ] Patch fixes verified in prod
- [ ] No release-blocking regressions observed

## 7) Rollback notes (required before release)

- [ ] Rollback command tested/documented with previous immutable tag
- [ ] Rollback owner identified
- [ ] Communication template prepared (rollback trigger + status update)

Rollback principle:

- Roll back by redeploying the previous known-good immutable tag.
- Do not roll forward with mutable tags in prod.

## 8) Release closeout

- [ ] Final semver tag created/pushed (`v3.0.0.1`) when prod is stable
- [ ] Release notes published
- [ ] Follow-up issues created for any deferred non-patch work
- [ ] Feature-track work remains tracked in [v3.1 QA checklist](./v3.1.md)
