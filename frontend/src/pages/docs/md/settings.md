# Settings

The Settings page (`/settings`) centralizes account/session controls, diagnostics toggles, and
legacy-save migration tools.

## Available panels in v3

- **Logout/credential controls** for local auth-related data
- **QA cheats toggle** (environment-gated)
- **Quest graph toggle** for quest diagnostics workflows
- **Legacy save upgrade** tools for v1 cookies and v2 localStorage migration
- **Data reset** controls

## Legacy migration support

Settings is the operational home for v1/v2 → v3 migration flows, including:

- detection banners
- merge vs replace import actions
- cleanup/discard options for old artifacts

For release-level migration verification, use the v3 QA checklist.
