DevOps in DSPACE is about keeping the stack healthy and deployable. The quests mirror
real-world practices: automated updates, CI pipelines, container deployments, and secure
HTTPS endpoints.

## What you will practice

- **Automation**: unattended upgrades, scheduled backups, and daily health checks.
- **CI/CD**: configuring pipelines that run tests and ship builds.
- **Containers**: launching DSPACE in Docker and validating service health.
- **Security**: TLS certificate renewal and HTTPS verification.

## Quest flow highlights

DevOps quests guide you through enabling automatic updates, setting up CI, configuring
backups, launching DSPACE in Docker, and securing the cluster with HTTPS.

## Inventory highlights

- Raspberry Pi cluster nodes, NVMe boot drives, and cooling cases.
- External backup SSDs and configuration snapshots.
- HTTPS service checks and certificate renewal logs.

## Why it matters

Reliable infrastructure keeps quests and data safe. DevOps practice reduces downtime,
prevents regressions, and ensures DSPACE can run anywhere.
