DSPACE v3 is a major content and platform release focused on three goals:

1. **Much more to do** (quests, processes, and items)
2. **Better creator workflows** (build and submit [custom content](/docs/custom-content) in-game)
3. **Safer, local-first player data** (migration, backups, and cloud sync)

If you are new to DSPACE, start with [About](/docs/about).

## v3 at a glance (vs v2.1)

The current v3 release snapshot includes:

- **248 quests** across **19 quest trees**
- **311 processes**
- **507 items**

Compared with v2.1 (current `main`):

| Category  | v2.1 | v3 | New in v3 |
| --------- | ---: | -: | --------: |
| Quests    | 22   | 248 | **+226** |
| Processes | 36   | 311 | **+275** |
| Items     | 90   | 507 | **+417** |

That means v3 delivers roughly **11.3x more quests** than v2.1, exceeding the original 10x target.

## Custom content (headline v3 feature)

v3 ships end-to-end creator tooling for player-generated content:

- [Custom quest creation](/quests/create), including validation and preview
- [Custom item creation](/inventory/create) (with `/items/create` preserved as a legacy alias)
- [Custom process creation](/processes/create), including requires/consumes/creates modeling
- [Bundle submission workflow](/bundles/submit) for grouped quest/item/process contributions

Like custom quests, your creations are stored locally by default. When you're ready to contribute,
package everything into a bundle following the [Custom Content Bundles](/docs/custom-content-bundles)
format and submit it through the in-game interface. This keeps related content synchronized and
makes reviews more efficient.

Reference docs:

- [Quest Submission Guide](/docs/quest-submission)
- [Custom Content Bundles](/docs/custom-content-bundles)

## Local-first saves, migration, and backups

v3 moves canonical save storage to IndexedDB and keeps migration from legacy v2 saves built in.

- On first launch, v3 detects legacy `localStorage` save keys and migrates data to IndexedDB.
- Save and content backup workflows are available in-game.
- Cloud backup and restore is available through GitHub Gist integration.

## AI chat in v3

v3 launches with **OpenAI-only chat** in production.

- NPC personas (such as Sydney, Nova, and Hydro) are available in chat.
- token.place integration is coming in v3.1, bringing free LLM usage instead of the metered
  usage that currently requires an OpenAI API key.

See:

- [v3 Release State](/docs/v3-release-state)
- [token.place rollout notes](/docs/token-place)

### dChat knowledge upgrade

dChat now ships with an in-game knowledge base that summarizes canonical quests, items, processes,
and the player's local inventory snapshot. This gives the assistant immediate context to answer
gameplay questions without waiting for future updates or external documentation lookups.

## Self-hosted

DSPACE 2.1 was migrated from Netlify to a local Mac Mini M4. DSPACE v3 now runs on-premises on a
k3s cluster across 9 Raspberry Pis.

## Switching to MIT License

While 0BSD sounds like a nice license, it is in some cases not on the list of accepted OSS licenses
on the Internet. Therefore, I'm switching from 0BSD to MIT, which is still very permissive but much
better supported.

## Looking Forward

These changes reflect a commitment to keeping DSPACE focused on its core mission: democratizing
space exploration through education, simulation, and real-world impact. By removing unnecessary
complexity, the game is more accessible while strengthening the connection between virtual
achievements and actual space development.

v3 is the biggest DSPACE update yet, and it lays the groundwork for faster content releases moving
forward.

## June 1, 2026 — DSPACE v3.0.1 (patch addendum)

`v3.0.1` is a patch update to the April 1 v3 release, focused on launch-readiness hardening across the audited delta `3ec45a5517a35c96767f6b946c01104e6ec88f93..899fcb93f705d0fe4051d7ecb74ee242cb8ab59c`.

### Patch notes

#### Gameplay / UI (`/quests`, `/processes`)

**What you'll notice**

- `/quests`: the list appears much faster, locked quest state renders correctly, and preflight-check quest flow no longer dead-ends.
- `/processes`: list rows render as summaries first, while runtime controls stay available on `/processes/:processId`.

**Details**

- Improved `/quests` list-path time-to-interactive and tightened built-in/custom coexistence behavior.
- Fixed locked-quest status visibility regressions.
- Switched `/processes` list rows to summary-first rendering and hardened buy-required metadata-loading behavior.

**Performance context (existing measured scenarios)**

- Cold run: `/quests` list visible 30.3 ms → 0.3 ms (~101x faster); full reconciliation pass 72.6 ms → 0.7 ms (~104x faster).
- 4x CPU-throttled run: `/quests` list visible 115.5 ms → 1.9 ms (~61x faster); full reconciliation pass 262.6 ms → 6.2 ms (~42x faster).

**How to verify**

- Open `/quests` on a fresh load and confirm the list appears quickly.
- Confirm locked quests show the correct locked status.
- Open `/processes` and confirm list rows render immediately as summaries, then open a `/processes/:processId` page and verify runtime controls are present.

#### Security / Safety

**What you'll notice**

- Custom quest tile links that used unsafe URL patterns now fall back to safe internal quest links instead of navigating to unsafe targets.

**Details**

- Sanitized custom quest tile routes to fall back to internal `/quests/{id}` links when values are unsafe, including protocol-relative prefixes like `//` and slash-backslash variants like `/\`.
- Sanitized compact item/process preview image handling.

**How to verify**

- Open quest tiles that use custom routes and confirm unsafe patterns (for example `//` or `/\`) resolve to internal `/quests/{id}` links.

#### Saves / Migration / Data Integrity

**What you'll notice**

- Saves with malformed item containers fail fast instead of being partially accepted, and migration/smoke coverage is more reliable for legacy-v2 paths.

**Details**

- Normalized stored item counts.
- Rejected unknown item containers.
- Fixed remote legacy migration real-v2 coverage and strengthened remote smoke flow reliability for custom-item mutations.

**How to verify**

- Run a legacy-v2 migration path and confirm it completes cleanly.
- Validate malformed saves/mod payloads with unknown containers are rejected instead of silently accepted.

#### Docs / Chat

**What you'll notice**

- `/docs` browsing and `has:` filters remain immediately available; first keyword search may trigger the deferred full-text corpus fetch.

**Details**

- Deferred `/docs` full-text corpus fetch until first keyword search, while keeping browse/`has:` access immediate.
- Hardened docs-index/chat smoke assertions and aligned geothermal reward docs with quest data.

**How to verify**

- Open `/docs` and confirm browse works immediately.
- Run a first keyword search and confirm index-backed search still returns expected results.
- Run the docs-backed chat smoke flow and confirm retrieval assertions pass.

#### Dev / Release / QA

**What you'll notice**

- No new player-facing feature surface here; this is release hardening and stronger verification coverage for the patch delta.

**Details**

- Tightened v3.0.1 QA checklists/evidence gates.
- Added canonical release-history tracking.
- Clarified rollback/promotion runbook links and compare-based PR references.
- Strengthened remote smoke flow reliability for live-chat checks.

#### Stricter behavior changes to be aware of

- Unsafe custom quest tile routes are now sanitized to internal `/quests/{id}` fallbacks.
- Unknown item containers are now rejected rather than accepted.

These are intentional hardening changes in `v3.0.1` and may expose previously malformed custom URLs or save/mod data earlier in the flow.
