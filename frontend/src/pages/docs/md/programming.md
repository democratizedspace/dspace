Programming quests teach you to capture sensor data, automate workflows, and publish results as
useful APIs. The focus is on real-world telemetry such as temperature logging.

## What you'll practice

- **Sensor input** — read thermistors and verify calibration.
- **Data logging** — store samples with timestamps and generate CSV exports.
- **APIs and alerts** — expose JSON endpoints, post updates, and send email alerts.
- **Visualization** — plot and graph temperature trends.

- Get data flowing: [Hello sensor](/quests/programming/hello-sensor),
  [Thermistor calibration](/quests/programming/thermistor-calibration),

  [Temp alert](/quests/programming/temp-alert)

## Inventory and tools to expect

Programming workflows reference:

- Arduino sensor rigs and thermistors
- Temperature log CSV exports and plotted charts

## Tips

- Validate sensors with a known reference before trusting automated alerts.
- Keep raw logs alongside derived metrics for later debugging.
