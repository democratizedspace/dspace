Programming quests build practical data workflows: collect sensor readings, clean logs, and
expose lightweight APIs for other systems.

## What you will practice

- **Data collection**: logging temperature readings and storing them in files.
- **Analysis**: computing averages and graphing time-series data.
- **APIs**: accepting POST data and serving JSON endpoints.
- **Visualization**: turning raw logs into readable charts.

## Quest flow highlights

Programming quests include logging temperature data, computing averages, graphing logs, and
accepting POST submissions. The output feeds into electronics, robotics, and monitoring
systems.

## Inventory highlights

- Arduino temperature logging rigs and thermistors.
- Laptops capable of running the tooling stack.
- Sample JSON endpoints and data log artifacts.

## Why it matters

Code connects sensors to decisions. Programming lets you turn raw telemetry into actions,
alerts, and automation.
